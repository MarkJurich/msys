Project History

Mingw32 was created by Colin Peters. The oldest preserved release is of 1998-07-01. Colin used Cygwin suite to compile code. First native mingw32 compiler was provided by Jan-Jaap van der Heijden. Besides GCC 2.8.1, his packages included GNU binutils, GNU make - i.e. everything to develop your own programs with Mingw32. Mumit Khan took the baton and over a couple of following years has been doing greate work of maintanance and development of Mingw32. He did many developments for supporting Win32-specific features in GCC and Binutils (which is of course of greate value for entire GNU/Win32), provided binary releases of them (with each new release of GCC and often before one was available for Cygwin) and developed more comprehensive bindings for runtime libraries. Significant addition to Mingw32 was Anders Norlander's w32api headers, which brought more comprehensive bindings for Win32 API that were available before.

In 199x, user base of Mingw32 became large enough to split off Cygwin mailing list and set up list specifically for Mingw32. Earnie Boyd and Dale Handerson moderated it.

At the beginning of Y2K, there was exposed public interest in development of Mingw, as well as offers for public services helping with OpenSource development, such as SourceForge, were made. It was decided to set up such a project for Mingw32 and call for maintainers and developers.
